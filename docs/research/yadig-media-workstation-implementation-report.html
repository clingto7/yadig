yadig 媒体工作站功能与本轮实现报告

本报告说明 yadig 当前支持的主要功能、使用方法、本轮 B 站媒体工作站实现内容、验证结果和下一步建议。

音乐发现 B 站同步 LLM 辅助分类 批量音频提取计划 本地资源库

一、当前项目定位

yadig 原本是音乐资源发现与信息聚合工具。本轮之后，项目开始向个人媒体信息、兴趣资源与音乐资料的资源管理工作站扩展。B 站相关能力是第一条垂直媒体链路，覆盖关注、收藏、稍后再看、元数据整理和音乐类视频音频提取。

搜索与发现

跨多个音乐与媒体源搜索艺术家、专辑、视频、乐评和资讯，并记录搜索历史。

资源沉淀

将搜索收藏、B 站收藏、关注和稍后再看沉淀到本地 SQLite 数据库中。

智能整理

通过 OpenAI-compatible LLM API 或本地 metadata fallback 生成分类、标签和操作建议。

二、当前支持的功能

跨源搜索

  • 支持 Pitchfork、Discogs、Bandcamp、Album of The Year、Jamendo、Stereogum、Fader、MusicBrainz、Last.fm、Bilibili、YouTube 等来源。
  • 支持分页加载、来源启停、搜索历史。
  • 可以将搜索结果加入本地收藏。

B 站音频提取

  • 支持 B 站视频 URL、短链和合集链接解析。
  • 支持单视频、多 P、章节切分和合集音频提取。
  • 使用 FFmpeg 对 B 站 DASH 音频做 remux，提高播放器兼容性。

YouTube 音频

  • 支持通过 yt-dlp 搜索 YouTube。
  • 支持 YouTube URL 音频提取。
  • 输出目录默认为 Downloads/yadig。

B 站登录

  • 支持二维码登录。
  • 支持完整 Cookie 导入，完整 Cookie 可提供写操作所需的 bili_jct 和 DedeUserID。
  • 支持账号密码登录入口，但遇到验证码时建议使用二维码或 Cookie。

媒体工作站

  • 同步 B 站收藏、关注 UP 和稍后再看。
  • 将同步结果保存到本地媒体库。
  • 支持 LLM 辅助分类、标签和操作建议。

批量音频计划

  • 从收藏和稍后再看的视频中筛出音乐类视频。
  • 生成批量音频提取计划。
  • 执行计划时复用已有 B 站音频提取链路。

三、使用方法

启动应用

  1. 安装依赖：pnpm install
  2. 开发运行：pnpm tauri dev
  3. 生产构建：pnpm build 或 pnpm tauri build

搜索与收藏

  1. 进入左侧导航的 Search。
  2. 输入艺术家、专辑、厂牌、风格、视频或事件关键词。
  3. 在结果中打开来源链接，或将感兴趣的条目加入本地收藏。
  4. 如果输入 B 站或 YouTube URL，页面会显示对应音频提取入口。

配置 B 站登录

  1. 进入 Settings。
  2. 在 Bilibili 区域选择 QR Code、Cookie Login 或 Account & Password。
  3. 推荐 QR Code 或完整 Cookie。只导入 SESSDATA 可以登录读取部分信息，但同步关注/收藏等能力需要 DedeUserID，批量管理类能力还需要 bili_jct。
  4. 登录状态会通过 B 站 nav API 校验，并显示普通账号或 Premium 状态。

配置 LLM

  1. 进入 Settings 的 API Keys 区域。
  2. 填写 Provider、Base URL、Model 和 API Key。
  3. 当前调用 OpenAI-compatible /chat/completions 接口，并要求模型返回 JSON。
  4. 没有 API Key 或调用失败时，工作站会使用本地 metadata fallback，并在界面提示 fallback 原因。

使用媒体工作站

  1. 进入左侧导航的 Workstation。
  2. 点击 Sync Bilibili 同步收藏、关注和稍后再看。
  3. 点击 Analyze Metadata 让 LLM 根据标题、分区、UP 信息等元数据生成分类。
  4. 点击 Create Plan 从音乐候选视频生成批量音频提取计划。
  5. 点击 Extract Audio 执行批量提取，输出文件在 Downloads/yadig。

五、数据与隐私说明

  • B 站 session 当前保存在运行时内存状态中，前端设置项保存在 Tauri Store 的 settings.json。
  • 媒体库、搜索历史、收藏、LLM 分析结果和操作计划保存在本地 SQLite 数据库 sqlite:yadig.db。
  • 执行 LLM metadata 分析时，条目的标题、作者、类型和 raw metadata 会发送给配置的 LLM API。
  • 没有配置 API Key 或 API 调用失败时，应用会使用本地规则 fallback，并提示 fallback 原因。
  • 批量音频提取只在用户确认后执行，输出到本机 Downloads/yadig。

七、已知限制与下一步

  • 当前 LLM 分析基于元数据，不读取视频字幕、简介全文或评论。
  • 字幕提取、无字幕音频转文字、视频要点总结尚未实现，是下一阶段自然扩展。
  • 微信、知乎等媒体源尚未接入，但本轮新增的本地媒体库模型可以作为后续扩展基础。
  • 批量管理目前重点是计划与音频提取，批量移动收藏夹、删除、归档等写操作还需要更细粒度确认和失败恢复设计。
  • 账号与 API Key 的长期安全存储可以后续升级到系统 keychain 或更细粒度加密方案。
